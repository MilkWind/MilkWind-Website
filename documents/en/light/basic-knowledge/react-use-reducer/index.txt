<h3 id="best-practices" class="group relative text-2xl font-semibold mt-5 mb-2 text-foreground leading-snug">
            <a href="#best-practices" class="absolute -left-6 top-0 opacity-0 group-hover:opacity-100 transition-opacity text-primary/60 hover:text-primary" aria-label="Link to Best Practices">
                <svg class="w-5 h-5" fill="currentColor" viewBox="0 0 20 20">
                    <path fill-rule="evenodd" d="M12.586 4.586a2 2 0 112.828 2.828l-3 3a2 2 0 01-2.828 0 1 1 0 11.414-1.414 4 4 0 105.656 0l3-3a4 4 0 00-5.656-5.656l-1.5 1.5a1 1 0 101.414 1.414l1.5-1.5zm-5 5a2 2 0 012.828 0 1 1 0 101.414-1.414 4 4 0 10-5.656 0l-3 3a4 4 0 105.656 5.656l1.5-1.5a1 1 0 10-1.414-1.414l-1.5 1.5a2 2 0 11-2.828-2.828l3-3z" clip-rule="evenodd" />
                </svg>
            </a>
            <strong class="font-bold text-foreground">Best Practices</strong>
        </h3>
<ol class="list-decimal list-inside mb-4 pl-6 space-y-2">
<li class="text-foreground leading-relaxed"><strong class="font-bold text-foreground">Pure functions</strong>: Reducer must be pure functions with no side effects</li>
<li class="text-foreground leading-relaxed"><strong class="font-bold text-foreground">Immutable updates</strong>: Always return new state objects, dont modify original state</li>
<li class="text-foreground leading-relaxed"><strong class="font-bold text-foreground">Action types</strong>: Use constants to define action types to avoid typos</li>
<li class="text-foreground leading-relaxed"><strong class="font-bold text-foreground">State structure</strong>: Keep state structure flat to avoid deep nesting</li>
<li class="text-foreground leading-relaxed"><strong class="font-bold text-foreground">Default case</strong>: Handle default case in reducer, usually return original state</li>
<li class="text-foreground leading-relaxed"><strong class="font-bold text-foreground">Error handling</strong>: Throw errors or warn for unknown action types</li>
</ol>
<h3 id="when-to-use-usereducer" class="group relative text-2xl font-semibold mt-5 mb-2 text-foreground leading-snug">
            <a href="#when-to-use-usereducer" class="absolute -left-6 top-0 opacity-0 group-hover:opacity-100 transition-opacity text-primary/60 hover:text-primary" aria-label="Link to When to Use useReducer">

                    <path fill-rule="evenodd" d="M12.586 4.586a2 2 0 112.828 2.828l-3 3a2 2 0 01-2.828 0 1 1 0 11.414-1.414 4 4 0 105.656 0l3-3a4 4 0 00-5.656-5.656l-1.5 1.5a1 1 0 101.414 1.414l1.5-1.5zm-5 5a2 2 0 012.828 0 1 1 0 101.414-1.414 4 4 0 10-5.656 0l-3 3a4 4 0 105.656 5.656l1.5-1.5a1 1 0 10-1.414-1.414l-1.5 1.5a2 2 0 11-2.828-2.828l3-3z" clip-rule="evenodd" />
                </svg>
            </a>
            <strong class="font-bold text-foreground">When to Use useReducer</strong>
        </h3>
<ul class="list-disc list-inside mb-4 pl-6 space-y-2">
<li class="text-foreground leading-relaxed"><strong class="font-bold text-foreground">Complex state logic</strong>: Multiple related state values need to be updated together</li>
<li class="text-foreground leading-relaxed"><strong class="font-bold text-foreground">Complex state transitions</strong>: Next state value depends on previous value</li>
<li class="text-foreground leading-relaxed"><strong class="font-bold text-foreground">Shared state across components</strong>: Combined with useContext for global state management</li>
<li class="text-foreground leading-relaxed"><strong class="font-bold text-foreground">Predictable state updates</strong>: Need clear state change history and debugging</li>
</ul>

            <strong class="font-bold text-foreground">Important Considerations</strong>
        </h3>
<ul class="list-disc list-inside mb-4 pl-6 space-y-2">
<li class="text-foreground leading-relaxed"><strong class="font-bold text-foreground">Performance considerations</strong>: Every dispatch triggers re-render</li>
<li class="text-foreground leading-relaxed"><strong class="font-bold text-foreground">Initialization cost</strong>: useReducer initialization is slightly heavier than useState</li>
<li class="text-foreground leading-relaxed"><strong class="font-bold text-foreground">Learning curve</strong>: Has some learning cost compared to useState</li>
<li class="text-foreground leading-relaxed"><strong class="font-bold text-foreground">Debugging complexity</strong>: State change logic is centralized in reducer, requires good debugging tools</li>
</ul>
